# Multiple Linear Regression from Scratch

Реализация множественной линейной регрессии с нуля на Python без использования готовых ML-библиотек.

---

## 📌 О проекте

В этом проекте реализована модель множественной линейной регрессии:

y = w1 * x1 + w2 * x2 + ... + b

Параметры модели обучаются с помощью градиентного спуска для минимизации функции ошибки (Mean Squared Error).

---

## ⚙️ Что реализовано

- Модель множественной линейной регрессии  
- Функция ошибки (MSE)  
- Градиентный спуск  
- Нормализация признаков (feature scaling)  
- Предсказание на новых данных  
- Эксперименты с learning rate  

---

## 🧩 Как работает модель

1. Данные нормализуются (каждый признак масштабируется отдельно)  
2. Инициализируются параметры модели (W и b)  
3. На каждой итерации:
   - считается предсказание: y = W · X + b  
   - считается ошибка  
   - вычисляются градиенты по W и b  
   - параметры обновляются через градиентный спуск  
4. После обучения модель используется для предсказания новых значений  

---

## 📊 Данные

Использовались реальные данные цен на квартиры:

- Признаки:
  - количество комнат  
  - площадь (м²)  

- Целевая переменная:
  - цена квартиры  

---

## 🔍 Feature Scaling

Сравнение обучения:

- ❌ Без нормализации → медленная и нестабильная сходимость  
- ✅ С нормализацией → быстрое и стабильное обучение  

**Вывод:**  
Feature scaling значительно ускоряет градиентный спуск и делает обучение более стабильным.

---

## 🔍 Learning Rate Experiments

Были протестированы разные значения learning rate (alpha):

- **alpha = 1** → самая быстрая сходимость (~700 итераций), но потенциально нестабильна  
- **alpha = 0.1** → стабильное обучение, требует больше итераций  
- **alpha = 0.01** → очень медленная сходимость  

**Вывод:**  
На текущем наборе данных `alpha = 1` показал наилучшую скорость обучения при сопоставимой ошибке, однако в общем случае может приводить к нестабильности.

## 🧠 Что я понял в процессе

- Модель обучается через минимизацию ошибки, а не “угадывает” значения  
- Градиент показывает направление изменения параметров  
- Learning rate влияет на скорость и стабильность обучения  
- Без нормализации градиентный спуск работает значительно хуже  
- Один набор весов используется для всех объектов (а не для каждого отдельно)
